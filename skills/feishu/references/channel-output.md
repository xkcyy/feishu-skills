# Feishu Channel Output

## Writing Style

- 简短、像同事沟通，不要写成说明书
- 能用短句就不用大段分点
- 先给结论，再补必要上下文

## Markdown Rules

- 飞书消息 Markdown 不是标准 Markdown 的完整子集
- 不确定时，先读 [lark-markdown.md](lark-markdown.md)
- 尤其注意标题层级、换行、@ 提及、图片 key、彩色文本和自定义标签

## Practical Advice

- 要在飞书消息里 `@` 用户时，优先先拿到准确 ID，再生成标签
- 要发图片时，不要假设 HTTP URL 可以直接放进消息 Markdown
- 复杂富文本和文档排版是两套规则
  消息输出看这个文件
  文档排版看 [docs.md](docs.md)
